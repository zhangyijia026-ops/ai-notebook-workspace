# AI 笔记本 MVP 验收 Checklist

## 1. 验收目标

确认当前 MVP 是否跑通核心闭环：

```text
记录笔记 / Todo
-> 保存摘录
-> AI 整理归类
-> 生成 Todo / 标签 / 摘要 / 分类
-> 转为笔记
-> 生成日报周报
-> 搜索找回
-> 本地保存 / 导出备份
```

## 2. 启动验收环境

在 `app/` 目录启动：

```bash
npm run dev:ai
npm run dev
```

验收地址：

- 前端应用：`http://127.0.0.1:5189/`，端口可能随 Vite 自动变化。
- AI 代理：`http://127.0.0.1:8787`，无需手动打开。

环境变量检查：

- [ ] `.env` 存在于 `app/` 目录。
- [ ] `VITE_AI_MODE=real`
- [ ] `VITE_AI_PROXY_URL=http://127.0.0.1:8787`
- [ ] `AI_API_KEY` 已配置。
- [ ] `AI_BASE_URL` 已配置为 One API 地址，例如 `https://oneapi.intelligrow.cn/v1`。
- [ ] `AI_MODEL` 已配置为 One API 可用模型。

## 3. 首页工作台

验收步骤：

- [ ] 打开应用后左上角显示“小张的笔记本”。
- [ ] 首页能看到未完成 Todo、今日 Todo、笔记、摘录数量。
- [ ] 使用“快速记录”保存为笔记。
- [ ] 使用“快速记录”保存为今日 Todo。
- [ ] 使用“快速记录”保存为摘录。

通过标准：

- [ ] 保存后对应模块数据数量增加。
- [ ] 刷新页面后数据仍存在。

## 4. 笔记本与笔记

验收步骤：

- [ ] 新建一个笔记本。
- [ ] 将笔记本重命名为“小张的笔记”或其他自定义名称。
- [ ] 给笔记本编辑描述。
- [ ] 在该笔记本中新建笔记。
- [ ] 编辑笔记标题和正文。
- [ ] 切换笔记所属笔记本。
- [ ] 删除一条笔记。
- [ ] 删除一个空笔记本。

通过标准：

- [ ] 有内容的笔记本可以重命名。
- [ ] 有内容的笔记本不能被误删。
- [ ] 空笔记本可以删除。
- [ ] 笔记编辑后自动保存，刷新页面仍存在。

## 5. 笔记内 Todo 同步

验收步骤：

在笔记正文中输入：

```markdown
- [ ] [高] [2026-05-27] 整理项目方案
- [ ] [中] 跟进用户反馈
- [x] [低] [2026-05-26] 归档会议纪要
```

通过标准：

- [ ] Todo 页面出现对应任务。
- [ ] 优先级被正确识别。
- [ ] 截止日期被正确识别。
- [ ] 已完成状态被正确识别。
- [ ] 从 Todo 可以跳转回来源笔记。
- [ ] 删除笔记中的 Todo 行后，Todo 页面同步更新。

## 6. Todo 管理

验收步骤：

- [ ] 新建 Todo。
- [ ] 设置高/中/低优先级。
- [ ] 设置截止日期。
- [ ] 标记完成。
- [ ] 取消完成。
- [ ] 使用筛选：今天、未完成、已完成、逾期、来自笔记。
- [ ] 关联来源笔记。

通过标准：

- [ ] Todo 状态、优先级、截止日期能即时保存。
- [ ] 刷新页面后 Todo 数据仍存在。
- [ ] 来源笔记跳转正确。

## 7. 摘录收集箱

验收步骤：

- [ ] 粘贴一条小红书内容。
- [ ] 来源选择“小红书”。
- [ ] 保存摘录。
- [ ] 粘贴一条 AI 聊天内容。
- [ ] 来源选择“AI 聊天”。
- [ ] 保存摘录。
- [ ] 按来源筛选摘录。
- [ ] 删除摘录。

通过标准：

- [ ] 摘录能保存原文、来源和可选链接。
- [ ] 刷新页面后摘录仍存在。
- [ ] 来源筛选结果正确。

## 8. AI 整理归类

验收步骤：

- [ ] 在笔记页选择一条有正文的笔记。
- [ ] 点击“AI 整理归类”。
- [ ] 检查是否生成标题、摘要、标签、推荐分类。
- [ ] 如果内容存在明确行动项，检查是否新增 Todo。
- [ ] 如果内容没有明确行动项，检查是否不新增 Todo。

通过标准：

- [ ] AI 标题不是“摘录/未命名/AI 对话”等泛称。
- [ ] AI 摘要能解释内容，而不是只复述原文。
- [ ] 标签适合后续检索。
- [ ] Todo 只在有明确任务时新增。

## 9. AI 草稿确认

验收步骤：

- [ ] 在收集箱选择一条摘录。
- [ ] 点击“AI 识别分类”。
- [ ] 检查是否出现“AI 草稿待确认”。
- [ ] 编辑建议标题。
- [ ] 编辑解读正文。
- [ ] 修改推荐笔记本。
- [ ] 修改标签。
- [ ] 修改 Todo。
- [ ] 点击“采纳并写入”。
- [ ] 再创建一条摘录，点击“AI 识别分类”后选择“放弃草稿”。

通过标准：

- [ ] AI 识别后不会直接写入 Todo。
- [ ] 采纳后才更新摘录摘要、标题、标签、推荐笔记本和 Todo。
- [ ] 放弃草稿不会改变摘录和 Todo。

## 10. 摘录转笔记

验收步骤：

- [ ] 对一条已采纳 AI 草稿的摘录点击“转为笔记”。
- [ ] 检查生成的新笔记标题。
- [ ] 检查新笔记正文结构。

通过标准：

- [ ] 新笔记默认使用 AI 建议标题。
- [ ] 正文包含：内容解读、可执行 Todo、可复用观点、归档信息、我的补充、原文。
- [ ] 新笔记进入推荐笔记本。

- [ ] 点击“生成日报”。
- [ ] 编辑日报内容。
- [ ] 保存到“日报周报”笔记本。
- [ ] 点击“生成周报”。
- [ ] 编辑周报内容。
- [ ] 保存到“日报周报”笔记本。

通过标准：

- [ ] 报告能基于 Todo、笔记、摘录生成。
- [ ] 报告可编辑。
- [ ] 保存后能在笔记本中找到。

## 12. 搜索

验收步骤：

- [ ] 搜索一个笔记标题关键词。
- [ ] 搜索一个 Todo 关键词。
- [ ] 搜索一个摘录原文关键词。
- [ ] 搜索一个 AI 摘要或标签关键词。
- [ ] 点击搜索结果。

通过标准：

- [ ] 搜索结果展示类型：笔记、Todo、摘录。
- [ ] 点击结果能跳转到对应页面。
- [ ] 笔记结果能打开对应笔记。
- [ ] Todo 来源笔记可以继续跳转。

通过标准：

- [ ] 应用说明当前数据保存在浏览器 localStorage。

- [ ] 导入前有覆盖提示。
- [ ] 导入后数据能恢复。
